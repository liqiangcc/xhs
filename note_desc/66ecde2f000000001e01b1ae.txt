Spring AOP
1.这个拦截器其实是一个 Spring AOP 思想的应用，简单介绍下 Spring AOP
Spring AOP（面向切面编程）允许在不修改业务逻辑的情况下将通用功能（如日志、事务管理、权限控制）切入到代码中。AOP通过切面（Aspect）和切入点（Pointcut）将特定的功能织入方法中，常见实现方式有JDK动态代理和CGLIB字节码增强。
2.Spring 已经用一些 Controller 支持了业务逻辑，为什么还要有 AOP 的存在？
AOP使得通用功能如日志、事务管理等与业务逻辑解耦，不需要在Controller中硬编码这些功能。AOP集中处理这些横切关注点，简化代码，减少重复，提升可维护性。
3.为什么用 ThreadLocal 类来封装用户信息，不能用一些简单的类来实现你这个功能吗？
ThreadLocal为每个线程提供独立的变量副本，适用于并发环境下的数据隔离。使用普通类在并发场景下会引发线程安全问题，而ThreadLocal确保每个线程有独立的用户信息，不会相互干扰。
Redis
4.简单说下 Redis 的底层数据结构
Redis的底层数据结构包括：
String：简单动态字符串或整数。
List：双向链表或压缩列表。
Set：无序集合，底层是哈希表或整数数组。
Sorted Set：跳表，用于按分数排序。
Hash：哈希表或压缩列表。
这些数据结构帮助Redis在不同场景下实现高效的存储和查询。
JVM
5.简单说下 JVM 的内存模型
JVM的内存模型分为以下区域：
堆（Heap）：存储对象实例，所有线程共享。
栈（Stack）：每个线程独有，存储局部变量和方法调用信息。
方法区（Method Area）：存储类信息、常量池、静态变量。
程序计数器（Program Counter）：记录每个线程当前执行的指令位置。
本地方法栈（Native Method Stack）：为本地方法服务，存储本地变量。
6.说一下 Java 虚拟机的类加载机制
JVM的类加载机制采用双亲委派模型：
加载：通过类加载器读取类的字节码。
链接：包括验证、准备、解析三个阶段。
初始化：对静态变量进行赋值，执行静态代码块。
双亲委派机制确保父类加载器优先加载类，避免核心类库被覆盖。
