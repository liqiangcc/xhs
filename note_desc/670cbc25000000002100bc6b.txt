所有问题大体上回答正确，约一周内二面挂。通过内部渠道得知，共有30+面试安排，只有2人进二面，除我之外另一个拒面等待秋招。截止八月中旬，该部门投递140+，0 offer。
被拒之后联系HR询问面评，略有争执（我问那个面试官的名字叫什么，HR不肯给，我就说他们面试流程不规范，没有任何自我介绍），然后好像还把我锁到飞书的部门去了，一个多月后才释放。
另外听说这个部门卡实习转正。这个消息不保真。
2⃣问题
只选取了非常规的题目，常规八股就不写了。
1. 代码运行结果判断：
for (int i = 0; i < 3; i++) {
Thread t = new Thread(() -> {
System.out.println(i);
});
t.start();
}
面试官认为正确结果应当是输出了三个3，因为在绝大多数情况下，三个线程会在非常接近的时间间隔内启动，此时指向的i的值为3.
然而实际上编译有可能不通过，因为Lambda表达式内部的变量会被修改。
2. Linux指令：
有一文件为广告曝光日志，提取出其中的有效部分，即ad_id和user_id。利用正则表达式匹配并处理即可。
3. 二面算法：
寻找一棵二叉树中两个节点的最近公共节点。每个节点存储着指向其父节点的指针，要寻找给定两个节点的最近的公共父节点。
实际上是Leetcode 相交链表的变式，二叉树在这里只起迷惑作用，寻找给定两个节点的最近公共父节点实际上就是找抽象出的两个链表的交点。
