🏗 项目拷打（滴滴最狠的部分）
1. 短链服务：高并发场景如何避免冲突，怎么快速反查？
* 哈希+Base62 编码，配合 分布式唯一 ID（Snowflake、Leaf 等）生成短链。
* 反查时依赖 Redis 缓存 + DB 持久化，提高 QPS。
2. 分布式环境下有没有考虑？
* 必问！要提 一致性（分布式锁 / CAS）和可扩展性（分库分表 + 缓存层）。
3. JWT 怎么保证安全？
* 使用 HMAC-SHA256 或 RSA 签名，设置过期时间，配合 Refresh Token
* 敏感信息不要放 JWT Payload。
4. 为什么用 MongoDB？
* JSON 文档存储方便，扩展性强，天然支持水平扩展。
* 但要提到 缺点（事务弱、复杂查询性能差），证明你考虑过 trade-off。
👉 反思：讲项目时不要死背，要结合高并发、分布式、缓存、容灾这些关键词，才显得你真的“想过”

⚙ Go 基础（高频必考）
* 变量声明：`var` 和 `:=` 的区别
* 切片是引用还是传值？（引用类型，但本质上传值传的是 slice header）
* slice 和 array 的区别
* map 并发安全？（默认不安全，用 `sync.Map` 或加锁）
* GPM 模型（Go 调度器核心）
* channel 使用与关闭
* 协程并发安全（锁、原子操作）
* context：取消 context 后，goroutine 不一定立即结束，需要主动检查 `<-ctx.Done()`
* defer执行顺序（LIFO）
* interface 和其他语言区别
👉 反思：这些知识点最好结合项目里的实际使用场景来回答，效果比死记硬背强 10 倍。
