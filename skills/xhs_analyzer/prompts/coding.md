# 💻 算法手撕_Coding 分析框架

> 适用：数据结构与算法题、手写代码题
> 典型题干：「算法：XXX」「手写 XXX」「实现 XXX」

## 输出结构

```markdown
## 💻 [原始题目]

### 1. 题意确认
- 输入是什么 / 输出是什么
- 边界条件和约束
- 需要和面试官确认的假设

### 2. 思路演进

**暴力解法**（O(?)）：
- 思路：...
- 为什么不够好：...

**优化解法**（O(?)）：
- 关键洞察：...（为什么可以优化）
- 核心数据结构 / 算法模式：...

### 3. 代码实现（Java）
// 附带关键行注释，解释 WHY 而非 WHAT

### 4. 复杂度分析
| | 时间 | 空间 |
|--|------|------|
| 暴力 | O(?) | O(?) |
| 优化 | O(?) | O(?) |

### 5. 测试用例
| 用例 | 输入 | 预期输出 | 测试目的 |
|------|------|----------|----------|
| 正常 | | | 验证基本逻辑 |
| 边界 | | | 空/单元素/最大值 |
| 特殊 | | | 重复/负数/溢出 |

### 6. 面试官追问预判 🔮
- 追问 1：如果数据量是 10 亿级呢？
- 追问 2：能否用其他数据结构？
- 追问 3：如何处理并发场景？
```

## 分析要求

- **代码必须可运行**：不是伪代码，是完整可编译的 Java
- **先暴力后优化**：展示思考过程，而非直接给最优解
- **注释解释 WHY**：关键行要注释"为什么这样写"而非"做了什么"
- **测试驱动**：先定义测试用例再写代码
