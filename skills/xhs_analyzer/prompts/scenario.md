# 🏗 场景设计_Scenario 分析框架

> 适用：系统设计、方案设计、高并发/高可用场景分析
> 典型题干：「如何设计 XXX 系统」「XXX 场景下如何保证 YYY」「场景：...」

## 输出结构

```markdown
## 🏗 [原始题目]

### 1. 需求拆解
**功能性需求**：
- ...

**非功能性需求**：
- 预估 QPS / 数据量 / 并发数
- 延迟要求 / 可用性要求

### 2. 核心架构方案
描述整体架构设计，包含核心组件和数据流。

### 3. 核心链路详设
**写链路**：请求入口 → 核心处理 → 数据落地
**读链路**：缓存策略 → 降级策略

### 4. 关键技术选型 & 理由
| 组件 | 选型 | 理由 | 备选方案 |
|------|------|------|----------|

### 5. 异常与容灾
| 故障场景 | 影响 | 应对策略 |
|----------|------|----------|
| 缓存击穿 | | |
| 数据库宕机 | | |
| 消息积压 | | |

### 6. 性能估算（Back-of-the-Envelope）
- 存储量 = ...
- 带宽 = ...
- 单机 QPS = ...

### 7. 面试官追问预判 🔮
- 追问 1：如果流量翻 10 倍呢？
- 追问 2：如何保证数据一致性？
- 追问 3：如何做灰度发布/回滚？
```

## 分析要求

- **量化**：所有估算必须给出具体数字和计算过程
- **取舍透明**：每个选型都要说清"为什么不选另一个"
- **故障驱动**：从"哪里会出问题"倒推设计
- **分层递进**：先给最简方案，再逐步演进到生产级
