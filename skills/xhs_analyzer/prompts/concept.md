# 🎯 八股文_Concept 分析框架

> 适用：知识点记忆、概念辨析、定义阐述
> 典型题干：「XXX 是什么」「XXX 有哪些」「XXX 和 YYY 的区别」

## 输出结构

```markdown
## 🎯 [原始题目]

### 1. 一句话结论
> 先说本质，不背定义。用一句话让面试官知道你理解核心。

### 2. 核心机制（2~4 点）
| 维度 | 要点 | WHY / 设计意图 |
|------|------|----------------|
只保留最重要的 2~4 个点，每个点说清"为什么这样设计"。

### 3. 工程取舍
- 为什么选它，不选相近方案
- 优点、代价、适用前提

### 4. 横向对比（如有近似概念）
| 对比项 | A | B |
|--------|---|---|
| 适用场景 | | |
| 优势 | | |
| 劣势 | | |

### 5. 线上场景 / 实践经验
- 这个概念在线上解决了什么问题
- 项目里怎么用过、踩过什么坑
- 常见误区是什么

### 6. 边界 / 风险 / 排障
- 什么情况下不适用
- 出问题怎么排查和兜底

### 7. 面试官追问预判 🔮
- 追问 1：...
- 追问 2：...
- 追问 3：...

### 8. 压缩版
- **1 分钟版**：结论 + 核心机制 + 一个取舍
- **项目版**：结论 + 线上场景 + 排障
- **追问版**：结论 + 取舍 + 边界
```

## 分析要求

- **数据化**：能给数字的给数字（如 "HashMap 默认容量 16，负载因子 0.75"）
- **结构化**：用表格对比而非大段文字
- **追问深度**：预判追问要比原题深一个层次（L1  L2）
- **禁止教材摘要**：每个知识点必须连接到工程判断，不要罗列定义

## 校招/实习降级规则

仅当用户明确为校招或实习时：
- 可在"一句话结论"后补充完整定义
- 可增加"记忆锚点"（类比或口诀）
- 可保留更多枚举式知识点拆解
