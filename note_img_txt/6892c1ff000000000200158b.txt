1. C 语言变量的生命周期

- 生命周期：变量从分配内存到释放内存的时间段。
- 全局 / 静态变量：程序运行全程有效（static 限定作用域，生命周期仍全局），存储在全局 / 静态存储区。
- 局部自动变量：进入作用域分配栈内存，离开作用域释放，存储在栈区。
- 动态分配变量：malloc 等分配后生效，free 或程序结束释放，存储在堆区。

可见性
- 可见性：变量能被代码访问的范围。
- 全局变量：工程内其他文件加 extern 可访问（跨文件可见）。
- 静态全局变量：仅定义所在文件可见（限制作用域）。
- 局部变量：仅所在函数 / 代码块可见。

2. 空指针和野指针的区别
- 空指针：显式赋值为 NULL（或 0）的指针，指向无效地址（通常是系统保留的 "空地址"，表示 "无指向"，是合法状态）。
- 野指针：指针未初始化、指向的内存已释放，或越界访问后形成的无效指针。指向的地址随机 / 非法，是 bug 导致的非法状态。

3. 访问野指针和空指针的后果
- 访问空指针：多数系统（如 Linux、RTOS）会触发段错误（Segmentation Fault），因为空地址通常无合法内存映射；嵌入式裸机中，若空地址未做特殊映射，可能访问到物理硬件（如寄存器），导致不可控行为。
- 访问野指针：行为不确定（UB，未定义行为）。可能读 / 写随机内存，导致程序崩溃、数据 corruption，或看似 "正常" 但埋下隐患（嵌入式中可能破坏硬件寄存器值，引发硬件异常）。

4. 释放后指针再次调用的问题与避免
- 问题：释放（free）后指针未置 NULL，再次解引用会触发 UB（同野指针，可能崩溃、破坏数据）；若指针被其他代码复用，还可能重复 free 导致堆损坏。
- 避免方法：
释放后立即置 NULL：free(ptr); ptr = NULL;, 后续调用前判断 if (ptr != NULL)。
用智能指针（C++ 嵌入式可选），或封装资源管理函数（如自定义 safe_free 统一处理）。

5. Nand Flash 基本原理
- Nand Flash 是嵌入式常用的非易失存储，存程序、数据。
- 靠浮栅晶体管存电荷表示 0/1，读按页（2KB~16KB）、写前要擦除块（块包含多页）擦除后才能写。
- 有坏块问题，要扫坏块记到坏块表，读写跳过，保证数据正确。

6. Nand Flash 初始化到使用流程（嵌入式典型步骤）
- 硬件初始化：配置 GPIO、时钟、SPI/EMMC 控制器等（如通过寄存器设置引脚复用、时序参数）。
- ID 读取与识别：发送命令读取厂商 ID、设备 ID，确认 Flash 型号，匹配驱动参数（页大小、块大小等）。
- 坏块扫描：遍历所有块，检测坏块并记录到 BBT（可在出厂时做，或运行时校验）。
- 读写操作：
    - 读：发送读命令 + 页地址，等待数据就绪后，从缓存寄存器读取。
    - 写：先擦除目标块（若需），发送写命令 + 页地址，将数据写入缓存寄存器，触发编程操作。
    - 擦除：发送擦除命令 + 块地址，触发块擦除，轮询状态判断完成。
- 高级功能：可选 ECC（错误校验与纠正，应对闪存位翻转）、磨损均衡（均衡块擦写次数，延长寿命）。

7. C++ 面向对象三要素、虚函数与多态
三要素：
- 封装：把硬件操作细节藏起来，对外给接口，比如传感器驱动类 Sensor，用 init()、read() 封装初始化和数据读取。
- 继承：子类继承父类，复用代码，像不同通信协议驱动继承 CommProtocol 基类，重写发送接收逻辑。
- 多态：同一接口，不同子类有不同实现，靠虚函数实现。

虚函数与多态：
- 基类声明 virtual 函数（如 virtual void processData()），子类可重写。
- 运行时，通过基类指针 / 引用调用，实际执行子类重写的版本（嵌入式中常用于不同硬件模块的统一抽象，如不同通信协议类 UARTProtocol、SPIProtocol 重写 sendData 虚函数）。

8. 进程间通信（IPC）方式
- 共享内存：多进程 / 任务映射同内存，直接读写，高效但要同步，比如嵌入式里多任务共享传感器数据缓冲区，用信号量锁保护。
- 消息队列：发消息异步通信，带优先级，适合任务间传指令，像系统事件通知。
- 信号量：做同步 / 互斥，保护共享资源（如硬件寄存器），或同步任务顺序。
- 套接字：支持网络 / 本地通信，跨设备（如嵌入式设备连服务器）或本地进程（AF_UNIX）都能用。

9. const int *p 和 int *const p 的区别
- const int *p：*p（指向的内容）只读，p（指针本身）能改指向。比如存硬件只读配置的指针，能换地址但不能改值。
- int *const p：p（指针本身）是常量，初始化后不能换指向，但 *p（内容）能改。像绑定某个硬件寄存器的指针，一直操作该寄存器，内容可配置。

10. 堆和栈的区别

维度 | 堆 | 栈
---|---|---
分配方式 | 编译器自动分配 / 释放 | 手动 malloc/free (或 C++ + new/delete )
空间管理 | 连续内存，栈顶生长（满则栈溢出，嵌入式中易因递归 / 大数组崩溃） | 离散分配，需手动管理（易内存泄漏、碎片）
空间大小 | 通常较小（由系统 / 编译器配置，如嵌入式可能仅几 KB~ 几十 KB） | 较大（受物理内存 / 交换空间限制，嵌入式中堆大下需谨慎配置）
存储内容 | 局部变量、函数参数、返回地址等 | 动态分配的变量（如大缓冲区、复杂数据结构）
访问效率 | 高（直接通过栈指针操作，硬件级支持） | 低（需遍历堆管理链表，分配释放有开销）

注：图中堆和栈的列标题疑似写反，常规 C/C++ 定义中：自动分配的是栈，手动分配的是堆。
