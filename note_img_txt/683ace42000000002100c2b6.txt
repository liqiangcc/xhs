知乎Java开发 技术二面- 1h30min 知乎

知乎面试中问到的题目：
1.自我介绍+项目
2.设计知乎的点赞系统
如何应对热点回答的瞬时高并发点赞？如何保证数据最终一致性
3.评论系统的性能优化
评论楼层展示如何避免 SELECT COUNT(*) 导致的性能问题?
4.分布式锁实现
用Redis实现一个可重入锁，如何解决锁续期问题？对比 Redisson 的实现。
5.热点Key问题
某大V的回答被频繁访问，如何避免Redis单节点过热?
6.手写线程安全LRU缓存
基于 LinkedHashMap 或自定义链表+哈希表实现。
7.如何优化Java序列化性能?
对比 Kryo、Hessian、Protobuf在知乎RPC框架中的选型。
8.JVM调优实战
知乎推荐服务频繁Full GC，如何定位并优化？ (G1 vs ZGC)
9.如何实现一个高性能的本地缓存?
对比 Caffeine 和 Guava Cache 的优劣。

10.CompletableFuture异步编排
如何实现 “查询用户信息+查询关注列表+计算相似度” 的并行调用？
11.如何排查Java应用CPU 100%问题?
结合 top -Hp + jstack 分析。
12.如何设计知乎的分布式ID生成器?
对比雪花算法、美团Leaf、数据库号段模式。
13.分布式事务方案
用户发布内容后，如何保证 “扣减积分+写入内容表+更新ES索引” 的一致性？
14.服务雪崩防护
如何用 Hystrix 或 Sentinel 实现接口熔断和降级？
15.RPC框架优化
如何设计一个支持动态负载均衡（如最小连接数）的RPC客户端
16.如何解决微服务下的跨库JOIN问题?
举例 “查询用户信息+用户最近回答”。
17.分库分表策略
知乎回答表按 answer_id 分片，如何高效查询某用户的全部回答

18.如何实现服务注册中心的高可用?
对比 Nacos 和 Zookeeper 的CP/AP特性。
19.如何设计一个灰度发布系统?
支持按用户ID、IP、设备类型分流。
20.如何保证MQ消息不丢失?
以Kafka为例，从生产者、Broker、消费者三方面分析。
21.如何实现接口幂等性?
举例 “用户重复提交回答” 的场景。
22.MySQL索引优化
如何优化 SELECT * FROM answers WHERE user_id=? AND create_time>? ORDER BY vote_count DESC?
23.如何解决幻读问题?
结合MVCC和间隙锁分析。
24.Redis持久化策略
知乎的评论缓存应该用RDB还是AOF？为什么？
25.如何设计一个高性能的排行榜?
基于 Redis ZSET 实现回答按热度排序。

26.分页查询优化
如何实现 “第10000页回答” 的高效查询？
27.如何优化大表DDL操作?
比如在千万级回答表上新增字段。
28.如何设计一个高可用的MySQL主从架构?
读写分离+故障转移方案。
29.如何避免缓存击穿?
举例 “某热门回答的缓存失效导致DB被打满” 。
30.如何优化JOIN查询?
知乎的 “用户-回答-评论” 多表关联查询如何设计索引？
31.如何设计一个高效的全文章检索系统?
基于 Elasticsearch 实现回答搜索。
32.如何设计知乎的Feed流系统?
推模式 vs 拉模式 vs 混合模式的选择。
33.如何设计一个实时在线人数统计系统?
基于 Redis HyperLogLog 或 布隆过滤器。
34.如何实现敏感词过滤?
基于 Trie树 或 DFA算法，支持动态更新词库。
