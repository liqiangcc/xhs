# 进程和线程的区别

## 核心结论

进程是操作系统资源分配的基本单位，线程是 CPU 调度执行的基本单位。进程之间地址空间隔离，线程共享同一进程内的内存和资源，所以线程更轻量，但也更容易出现并发安全问题。

## 1 分钟版

进程有独立虚拟地址空间、文件描述符、堆栈等资源，进程间通信通常需要管道、消息队列、共享内存、Socket 等 IPC 机制。线程属于进程，同一进程内的线程共享堆、方法区、文件句柄等资源，每个线程有自己的栈、程序计数器和寄存器上下文。创建和切换线程通常比进程轻量，但线程共享内存，必须处理锁、可见性、死锁等并发问题。

## 3 分钟版

可以从资源、调度、隔离、通信、开销五个角度回答。第一，进程拥有独立地址空间，是资源隔离边界；线程共享进程地址空间，是执行流。第二，操作系统调度时实际调度的是线程，进程更像资源容器。第三，一个进程崩溃通常不直接破坏其他进程，而一个线程异常可能导致整个进程退出。第四，进程通信需要 IPC，线程通信可以直接读写共享变量，但要保证同步。第五，进程创建和上下文切换要切换页表、地址空间等，线程切换主要切换寄存器和栈，成本更低。

## 关键细节

- 进程隔离强，适合故障隔离和安全隔离。
- 线程共享资源，适合高并发任务拆分和 CPU/IO 并行。
- 多线程性能收益来自减少创建/切换成本和共享数据成本。
- 多线程风险集中在竞态条件、内存可见性、死锁和资源泄漏。

## 原理机制

- 进程上下文包含虚拟内存、页表、文件描述符、信号处理等。
- 线程上下文包含线程栈、寄存器状态、程序计数器和调度状态。
- Linux 中线程常由轻量级进程实现，本质是共享部分资源的 task。

## 项目经验版

在线服务通常使用一个进程承载多个工作线程，例如 Web 容器线程池或 RPC 线程池。进程用于隔离服务实例，线程用于并发处理请求。遇到 CPU 密集型任务时要限制线程数，避免频繁上下文切换；遇到 IO 密集型任务时可以用线程池或异步模型提高吞吐。

- 多线程一定比多进程快吗？
- 线程共享哪些资源，不共享哪些资源？

## 易错点

- 不要说进程是 CPU 调度基本单位，现代系统通常调度线程。
- 不要只说“线程共享内存”，还要说明共享带来的同步问题。
- 不要忽略进程隔离带来的稳定性优势。
