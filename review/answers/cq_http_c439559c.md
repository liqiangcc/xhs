# HTTP 和 HTTPS 的区别

## 核心结论

HTTPS 是在 HTTP 和 TCP 之间加入 TLS 安全层后的协议组合。它相比 HTTP 提供加密传输、身份认证和完整性校验，但会增加握手和证书管理成本。

## 1 分钟版

HTTP 明文传输，内容可能被窃听、篡改或伪造。HTTPS 使用 TLS 对通信加密，通过证书验证服务端身份，并用消息认证机制保证数据完整性。常见端口上 HTTP 是 80，HTTPS 是 443。HTTPS 首次连接需要 TLS 握手，现代协议通过会话复用、TLS 1.3、HTTP/2 等方式降低额外开销。

## 3 分钟版

HTTPS 的握手过程大致包括客户端发送支持的协议和随机数，服务端返回证书、随机数和协商参数，客户端校验证书后完成密钥协商，后续用对称加密传输业务数据。非对称加密主要用于身份认证和密钥协商，对称加密负责高效传输。证书链由 CA 体系保证可信，浏览器会校验证书域名、有效期、签名链和吊销状态。对后端系统来说，HTTPS 还涉及证书续期、网关卸载、内网链路是否加密等架构问题。

## 关键细节

- HTTP 是明文，HTTPS 通过 TLS 加密。
- HTTPS 可以防窃听、防篡改，并验证服务端身份。
- TLS 握手会增加连接建立成本。
- 证书过期、域名不匹配、链不完整都会导致访问失败。

## 原理机制

- 非对称加密用于认证和协商密钥。
- 对称加密用于后续高性能数据传输。
- 摘要和 MAC 机制保证完整性。
- CA 证书链提供信任根。

## 项目经验版

生产系统通常在负载均衡或网关层终止 HTTPS，再转发到后端服务。对外接口必须配置证书监控和自动续期，避免证书过期事故。涉及支付、登录、个人信息的内网链路，也要评估是否需要端到端加密或服务间 mTLS。

- HTTPS 一定安全吗？

## 易错点

- 不要说 HTTPS 只是把端口从 80 换成 443。
- 不要认为 HTTPS 可以防止所有攻击。
- 不要忽略证书管理和 TLS 版本配置。
