# 搜索引擎设计:信息搜索怎么避免大海捞针?

## 核心结论

搜索引擎避免“大海捞针”的核心是建立倒排索引，把“文档里有什么词”转换成“词在哪些文档中出现”，查询时先召回候选文档，再通过相关性打分、过滤、排序和分页返回结果。

## 1 分钟版

如果直接扫描所有文档，数据量大时不可接受。搜索系统会先对内容分词、归一化，建立 term -> doc list 的倒排索引。查询时也分词，然后在倒排表中快速找到包含关键词的文档集合，再根据 TF-IDF、BM25、字段权重、业务权重等打分排序。为了提高效果，还会做同义词、拼写纠错、过滤条件、缓存和分片。

## 3 分钟版

整体链路包括写入建索引和查询检索。写入时对标题、正文、标签等字段分词，记录词项、文档 ID、词频、位置等信息，形成倒排索引。查询时对用户输入分词，做布尔查询、短语查询或多字段查询，从倒排索引中召回候选集。然后进行相关性排序，结合文本相关度、时间、热度、点击率、权限等业务因素。大规模场景下索引会分片，查询并发到多个 shard，再做结果合并。为了保证性能，还会缓存热门查询和控制深分页。

## 关键细节

- 倒排索引是核心结构。
- 分词质量直接影响召回率和准确率。
- 相关性排序通常结合文本分和业务分。
- 深分页、热点查询、索引更新延迟都要处理。

## 原理机制

- 正排索引是 doc -> terms。
- 倒排索引是 term -> docs。
- BM25 会考虑词频、文档长度和逆文档频率。
- 分片让索引和查询可以水平扩展。

## 项目经验版

设计商品或内容搜索时，我会把标题、类目、标签、描述同步到 Elasticsearch。查询侧根据关键词和筛选条件召回，再叠加销量、时间、质量分排序。对热门词做缓存，对运营配置词做人工干预，对权限或状态字段做 filter。

## 易错点

- 不要只说“用 ES”，要讲倒排索引和召回排序。
- 不要忽略分词和相关性打分。
- 不要把搜索引擎当强一致事务库。
