# MySQL 7 天复习计划

> 适用对象：希望在 7 天内把 MySQL 高频面试题从“看过”提升到“能讲清楚”。
> 复习材料主入口：`mysql_questions.md` + `mysql_review_plan.md` + 两份专题 session。
> 执行原则：**先母题、后变体；先主干、后长尾；先能讲、后细抠。**

## 使用方式

| 时间块 | 建议占比 | 做什么 |
|---|---|---|
| 主干知识梳理 | 40% | 用知识链把同类题串起来，形成 1 份口述答案 |
| 真题抽问训练 | 40% | 从题库里抽题，强制口头回答 |
| 复盘与错题整理 | 20% | 记录卡顿点、补追问、第二天复习 |

**每日统一动作：**
- 先花 15 分钟复盘昨天卡住的 5~10 个点。
- 每天只攻 2 个模块，避免在 631 题里横跳。
- 每道题至少产出 3 个必答点 + 2 个追问点。
- 每晚做一次 20 分钟模拟抽问，只讲不看答案。

- undo log 版本链如何支撑 MVCC
- RC 和 RR 下 Read View 的区别

**重点材料：**
- `mysql_questions.md` 中的“事务与锁”主题
- `transaction_lock_review_session.md`

**今天必须讲顺的 6 个问题：**
- ACID 在 MySQL 里分别靠什么机制保证？
