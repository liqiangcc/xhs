# 面试题库系统 — 基于 Agent Skill 的重构方案

## 核心痛点与思路转换

你的思路非常超前：“不要固化模型和脚本，使用 Agent Skill 技术”。
**以前的思路（传统研发）**：写 Shell 脚本调用 Gemini CLI，写 Python 脚本去重，写 Python CLI 交互。**缺点**：API 和模型被硬编码、脚本可维护性差、死板。
**现在的思路（Agent 化）**：不写死代码，而是定义两套 **Agent Worflow/Skill**，让 AI Agent（我）作为操作者。数据处理和面试辅导变成我（大模型）的两个“技能”。你只需要在对话框里对我下指令即可。

---

## 阶段拆解与 Agent Skill 设计

我们将把系统抽象为两个核心能力，作为 `.agents/workflows/` 下的两个标准 Agent 技能。

### Skill 1: 智能知识萃取 (Data Structuring Agent)

**目标**：将杂乱的原始面经转化为标准化的题库。完全由 Agent 动态调用工具完成，不写死抽取逻辑和模型。

**机制设计**（未来将写成 `.agents/workflows/extract_questions.md`）：
1. **触发方式**：你对我说 `执行面试题数据萃取`。
2. **Agent 行为**：
   - 自动遍历 `note_desc/` 目录。（使用 `list_dir` / `view_file` 工具）
   - 基于语言模型的强大理解能力，动态识别公司、岗位、题目。（替代死板的正则是提取）
   - 直接通过工具将结构化结果写入 `note_structured/` 的 JSON 文件。
3. **优势**：换模型？不需要改代码。换解析规则？只需改提示词。

### Skill 2: 全真模拟面试官 (Interactive Tutor Agent)

**目标**：抛弃繁琐的 CLI 和网页，将当前的 IDE/对话框直接变成沉浸式的“面试沟通界面”。

**机制设计**（未来将写成 `.agents/workflows/interview_tutor.md`）：
1. **触发方式**：
   - “开启高频题模式（针对百度）” -> Agent 会自动读取清洗后的数据，挑出高频题按顺序提问。
   - “开启专项攻克模式（JVM）” -> Agent 过滤出 JVM 题目对你提问。
2. **交互循环模式 (Agent 工作流)**：
   - **Agent 出题**：从题库中抽取一道未掌握的题目发给你。
   - **你回答**：你在对话框里输入、口述记录，或者仅仅在心里默念然后回复“我会了/需要提示/完全不懂”。
   - **Agent 深度解析与点评**：Agent 利用其内在的强大知识库，对题目进行“一句话破题 -> 原理深度剖析 -> 面试官连环追问预测”。如果你给出了答案，Agent 还会像真实面试官一样打分纠错。
   - **状态记录**：Agent 根据你的掌握程度，更新题库状态（例如打上标签 `掌握程度: 弱`），实现类似 Anki 的记忆追踪。

## 整体实施路径 (Action Plan)

如果采用 Agent Skill 架构，我们需要做的是 **定义规则（编写 Prompt 流程），而非编写死板的代码**。

### Phase 1: 编写知识萃取 Skill
- 创建系统工作流文件：`workflows/extract_xhs_questions.md`
- 用一小批数据（比如 10 个 note）让 Agent 进行跑通测试，验证其能否漂亮地聚合相似题目，并产出高质量结构化文件 `question_bank.json`。

### Phase 2: 编写面试官 Skill
- 创建系统工作流文件：`workflows/interview_tutor.md`
- 设定面试官的 Prompt 调性（严厉/和蔼/深度挖掘），以及阅读本地 JSON 题库的出题轮询逻辑。

### Phase 3: 直接开始刷题演练
- 你只需要一句：`/interview_tutor focus="并发编程" company="美团"`，你和我的对话就正式进入沉浸式刷题状态。
