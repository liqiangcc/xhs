# 面试题库系统 — 核心用例 (Use Cases)

> Historical note: this document is archived for context. The current execution plan lives in `docs/refactor/`.

为了不被具体的技术实现（无论是脚本还是 Agent Skill）局限，我们先纯粹从 **使用者的视角**，明确你在备战面试时，究竟会怎么“用”这个系统。

基于我们之前的沟通，核心聚焦于 **无缝刷题** 和 **深度解析**，主要划分为以下两个核心用例：

---

## 用例 1: 按特征全景浏览 (对应你的“场景 A”)
**目标**：掌握目标公司或整体大盘的高频考点，做到“心里有数”。

* **触发条件**：你想要了解最近各大厂，或者特定某家大厂（如百度）都在重点考察什么。
* **主要流程**：
  1. **输入指令**：你向系统提出需求，例如：“列出百度的前 10 个最高频面试题” 或 “列出现在 Java 后端整体最高频的 20 道题”。
  2. **系统检索**：系统从底层的题库数据中，根据公司、岗位过滤，并按出现频率倒序排列题目。
  3. **系统展示**：系统给出一个清晰的清单（题目 + 频次 + 来源）。
  4. **深入（可选）**：你在看清单时，对某道题（例如“ConcurrentHashMap 如何解决线程安全”）不熟悉。你对系统说：“给我解析第 3 题”。
  5. **AI 深度解析**：系统立刻扮演资深面试官，给出该题的：
     - 一句话核心本质
     - 分点深度解答（带源码解析或设计权衡）
     - 面试官可能会接着追问的 2-3 个连环问题。

---

## 用例 2: 按知识点专项突破 (对应你的“场景 C”)
**目标**：在复习特定模块（如 JVM、并发、MySQL）时，通过高频真实面试题进行“以战代练”。

* **触发条件**：你今天计划专门复习并攻克 "JVM" 模块。
* **主要流程**：
  1. **输入指令**：你向系统提出需求，例如：“开启 JVM 专项复习模式，给我按频率推题”。
  2. **系统出题**：系统从题库中找出属于 JVM 分类下的最高频题目，并将**第一道题**发送给你（例如：“讲讲 CMS 的回收过程，会 Stop The World 几次？”）。
  3. **用户作答（或跳过）**：
     - **尝试解答**：你在对话框里简要写出你的答案关键点。
     - **请求直接看答案**：或者你直接说“我不会，给标准答案”。
  4. **AI 点评与深度解析**：
     - 如果你做了回答，系统会指出你回答的亮点和缺失点，并给出一个面试能拿高分的“标准表述”。
     - 无论你是否回答，系统都会像用例 1 一样，提供深度解析和预测追问。
  5. **状态流转**：系统询问：“这道题掌握得如何？”。你标记（如：完全掌握 / 模糊 / 不熟悉）。
  6. **继续推进**：系统自动推送本模块频率排在第二的题目，循环直至你想停止。

---

## 用例 3: 隐式用例 - 数据清洗与题库构建 (后台用例)
**目标**：支撑前台高效率检索，将杂乱数据化为高质量题库。

* **触发条件**：你有新爬取的 XHS 面经数据放入了指定文件夹。
* **主要流程**：
  1. 系统读取原始面经的乱码文本。
  2. 系统自动识别每篇面经的公司、岗位、轮次。
  3. 系统准确剥离出纯净的面试题目列表，剔除杂音和非面试题内容。
  4. 系统跨文件对比，将含义相同但表述不同的题目（如“说说 HashMap 扩容” 和 “HashMap 什么时候扩容”）合并计算频次。
  5. 系统自动为合并后的题目打上知识模块标签（如 JVM、MySQL、中间件）。
